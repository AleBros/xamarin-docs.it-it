# <a name="controls-reference"></a>Riferimento per i controlli

Xamarin. Forms Usa utilizza controlli nativi di ciascuna piattaforma di destinazione.
Ciò significa che le applicazioni possono utilizzare [librerie di classi portabili](~/cross-platform/app-fundamentals/pcl.md) o [progetti condivisi](~/cross-platform/app-fundamentals/shared-projects.md) per ospitare il codice condiviso e quindi applicazioni specifiche della piattaforma che utilizzano il codice condiviso.

Esistono quattro gruppi di controllo principale utilizzati per creare l'interfaccia utente di un'applicazione di xamarin. Forms:

- [**Pagine**](pages.md)
- [**Layouts**](layouts.md)
- [**Visualizzazioni**](views.md)
- [**Celle**](cells.md)

In fase di esecuzione che l'equivalente nativo verrà mappato ogni controllo, che è ciò che verrà eseguito il rendering.

In questa sezione funge da riferimento che illustra molti dei controlli dell'interfaccia utente disponibili per l'uso con xamarin. Forms e corredata di [FormsGallery esempio](https://developer.xamarin.com/samples/FormsGallery/), che illustra l'uso di base di ogni controllo.
Ogni descrizione anche collegamenti al [documentazione dell'API](https://developer.xamarin.com/api/namespace/Xamarin.Forms/).
