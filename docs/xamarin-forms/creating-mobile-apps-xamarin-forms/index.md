# <a name="creating-mobile-apps-with-xamarinforms-book"></a>Creazione di app per dispositivi mobili con Xamarin.Forms book

[![Scaricare ](~/media/shared/download.png) l'esempio scaricare l'esempio](https://github.com/xamarin/xamarin-forms-book-samples)

<img src="images/cover-sml.png" title="Creazione di app per dispositivi mobili con Xamarin.Forms book" align="left" />Il libro *creazione di app per Xamarin.Forms dispositivi mobili con* Charles Petzold è una guida per imparare a scrivere Xamarin.Forms applicazioni. L'unico prerequisito è la conoscenza del linguaggio di programmazione C#. Il libro fornisce un'esplorazione approfondita dell' Xamarin.Forms interfaccia utente e include anche animazioni, MVVM, trigger, comportamenti, layout personalizzati, renderer personalizzati e molto altro ancora.

Il libro è stato pubblicato nella primavera del 2016 e non è stato aggiornato da allora. C'è molto del libro che rimane prezioso, ma parte del [materiale è obsoleta](#ways-in-which-the-book-is-outdated)e alcuni argomenti non sono più completamente corretti o completi.

## <a name="ways-in-which-the-book-is-outdated"></a>Modi in cui il libro è obsoleto

Dalla pubblicazione della *creazione di app per dispositivi Xamarin.Forms mobili con *, sono state aggiunte diverse nuove funzionalità a Xamarin.Forms . Queste nuove funzionalità sono descritte in singoli articoli della [Xamarin.Forms](/xamarin/) documentazione.

Altre modifiche hanno causato la scadenza di alcuni contenuti del libro:

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>.NET Standard librerie 2,0 hanno sostituito le librerie di classi portabili

Un' Xamarin.Forms applicazione usa in genere una libreria per condividere il codice tra le diverse piattaforme. Originariamente, era una libreria di classi portabile (PCL). Ci sono molti riferimenti a classi portabili nel libro e i riepiloghi del capitolo.

La libreria di classi portabile è stata sostituita da una libreria di .NET Standard 2,0, come descritto nell'articolo [supporto Xamarin.Forms di .NET standard 2,0 in ](~/xamarin-forms/internals/net-standard.md). Tutto il [codice di esempio](https://github.com/xamarin/xamarin-forms-book-samples) del libro è stato aggiornato per usare le librerie di .NET standard 2,0.

La maggior parte delle informazioni del libro relative al ruolo della libreria di classi portabile rimane invariata per una libreria .NET Standard 2,0. Una differenza è che solo una libreria di classi portabile ha un "profilo" numerico. Sono inoltre disponibili alcuni vantaggi per le librerie .NET Standard 2,0. Il capitolo 20, [Async e i/o di file](summaries/chapter20.md) , ad esempio, descrive come usare le piattaforme sottostanti per l'esecuzione di operazioni di i/o di file. Ciò non è più necessario. La libreria .NET Standard 2,0 supporta le classi [System.io](xref:System.IO) note per tutte le Xamarin.Forms piattaforme.

La libreria .NET Standard 2,0 consente inoltre alle Xamarin.Forms applicazioni di utilizzare [`HttpClient`](xref:System.Net.Http.HttpClient) per accedere ai file su Internet anziché [`WebRequest`](xref:System.Net.WebRequest) o ad altre classi.

### <a name="the-role-of-xaml-has-been-elevated"></a>Il ruolo di XAML è stato elevato

*Creazione di app per Xamarin.Forms dispositivi mobili con* inizia con la descrizione della scrittura di Xamarin.Forms applicazioni con C#. Il Extensible Application Markup Language (XAML) non è stato introdotto fino al [capitolo 7. XAML rispetto al codice](summaries/chapter07.md).

XAML dispone ora di un ruolo molto più grande in Xamarin.Forms . I Xamarin.Forms modelli di soluzione distribuiti con Visual Studio creano file di paging basati su XAML. Uno sviluppatore Xamarin.Forms che usa deve acquisire familiarità con XAML il prima possibile. La sezione [Extensible Application Markup Language (XAML)](~/xamarin-forms/xaml/index.yml) della Xamarin.Forms documentazione contiene diversi articoli su XAML per iniziare.

### <a name="supported-platforms"></a>Piattaforme supportate

Xamarin.Formsnon supporta più Windows 8.1 e Windows Phone 8,1.

Il libro a volte fa riferimento al _Windows Runtime_. Si tratta di un termine che comprende l'API Windows usata in diverse versioni di Windows e Windows Phone. Le versioni più recenti di Xamarin.Forms si limitano a supportare il piattaforma UWP (Universal Windows Platform), ovvero l'API per Windows 10 e Windows 10 Mobile.

Una libreria .NET Standard 2,0 non supporta alcuna versione di Windows 10 Mobile. Pertanto, un' Xamarin.Forms applicazione che utilizza una libreria di .NET standard non viene eseguita in un dispositivo Windows 10 Mobile. Xamarin.Formsle applicazioni continuano a essere eseguite in Windows 10 desktop, versioni 10.0.16299.0 e successive.

Xamarin.Formsdispone del supporto in anteprima per le piattaforme [Mac](~/xamarin-forms/platform/other/mac.md), [WPF](~/xamarin-forms/platform/other/wpf.md), [GTK #](~/xamarin-forms/platform/other/gtk.md)e [Tizen](~/xamarin-forms/platform/other/tizen.md) .
