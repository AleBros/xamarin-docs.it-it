# <a name="creating-mobile-apps-with-xamarinforms-book"></a>Creazione di app per dispositivi mobili con il libro Xamarin.Forms

[![Scarica](~/media/shared/download.png) l'esempio Scarica l'esempioDownload Sample Download the sample](https://github.com/xamarin/xamarin-forms-book-samples)

<img src="images/cover-sml.png" title="Libro Creazione di app per dispositivi mobili con Xamarin.Forms" align="left" />Il libro *Creating Mobile Apps with Xamarin.Forms* di Charles Petzold è una guida per imparare a scrivere applicazioni Xamarin.Forms. L'unico prerequisito è la conoscenza del linguaggio di programmazione C . Il libro fornisce un'ampia esplorazione nell'interfaccia utente di Xamarin.Forms e copre anche l'animazione, MVVM, trigger, comportamenti, layout personalizzati, renderer personalizzati e molto altro ancora.

Il libro è stato pubblicato nella primavera del 2016 e da allora non è stato aggiornato. C'è molto nel libro che rimane prezioso, ma parte del [materiale è obsoleto,](#ways-in-which-the-book-is-outdated)e alcuni argomenti non sono più del tutto corretti o completi.

## <a name="ways-in-which-the-book-is-outdated"></a>Modi in cui il libro è obsoleto

Dalla pubblicazione di *Creating Mobile Apps with Xamarin.Forms*, sono state aggiunte diverse nuove funzionalità a Xamarin.Forms. Queste nuove funzionalità sono descritte in singoli articoli nella documentazione di [Xamarin.Forms.These](/xamarin/) new features are described in individual articles in the Xamarin.Forms documentation.

Altre modifiche hanno causato alcuni dei contenuti del libro obsoleti:

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>Le librerie .NET Standard 2.0 hanno sostituito le librerie di classi portabili

Un'applicazione Xamarin.Forms utilizza in genere una libreria per condividere il codice tra le diverse piattaforme. Originariamente, si trattava di una libreria di classi portabile (PCL). Ci sono molti riferimenti ai PCL in tutto il libro e i riepiloghi dei capitoli.

La libreria di classi portabile è stata sostituita con una libreria .NET Standard 2.0, come descritto nell'articolo [Supporto di .NET Standard 2.0 in Xamarin.Forms](~/xamarin-forms/internals/net-standard.md). Tutto il [codice di esempio](https://github.com/xamarin/xamarin-forms-book-samples) del libro è stato aggiornato per utilizzare le librerie .NET Standard 2.0.

La maggior parte delle informazioni contenute nel libro relative al ruolo della libreria di classi portabile rimane la stessa per una libreria .NET Standard 2.0. Una differenza è che solo una libreria di classi Portaiata ha un "profilo" numerico. Inoltre, esistono alcuni vantaggi per le librerie .NET Standard 2.0.Also, there are some advantages to .NET Standard 2.0 libraries. Ad esempio, Chapter 20, [Async e File I/O](summaries/chapter20.md) descrivono come utilizzare le piattaforme sottostanti per l'esecuzione di I/O di file. Ciò non è più necessario. La libreria .NET Standard 2.0 supporta le classi [System.IO](xref:System.IO) familiari per tutte le piattaforme Xamarin.Forms.

La libreria .NET Standard 2.0 consente inoltre alle [`HttpClient`](xref:System.Net.Http.HttpClient) applicazioni Xamarin.Forms [`WebRequest`](xref:System.Net.WebRequest) di accedere ai file tramite Internet anziché su altre classi.

### <a name="the-role-of-xaml-has-been-elevated"></a>Il ruolo di XAML è stato elevato

La creazione di *app per dispositivi mobili con Xamarin.Forms* inizia descrivendo come scrivere applicazioni Xamarin.Forms utilizzando C . Il linguaggio XAML (Extensible Application Markup Language) non viene introdotto fino al [capitolo 7. XAML e codice](summaries/chapter07.md).

XAML now has a much larger role in Xamarin.Forms. I modelli di soluzione Xamarin.Forms distribuiti con Visual Studio creano file di paging basati su XAML. Uno sviluppatore che usa Xamarin.Forms deve acquisire familiarità con XAML il prima possibile. La sezione [XAML (Application Markup Language) eXtensible](~/xamarin-forms/xaml/index.yml) della documentazione di Xamarin.Forms contiene diversi articoli su XAML per iniziare.

### <a name="supported-platforms"></a>Piattaforme supportate

Xamarin.Forms non supporta più Windows 8.1 e Windows Phone 8.1.

Il libro a volte fa riferimento a _Windows Runtime_. Si tratta di un termine che include l'API di Windows utilizzata in diverse versioni di Windows e Windows Phone. Versioni più recenti di Xamarin.Forms si limita a supportare la piattaforma Windows universale, che è l'API per Windows 10 e Windows 10 Mobile.

Una libreria .NET Standard 2.0 non supporta alcuna versione di Windows 10 Mobile. Pertanto, un'applicazione Xamarin.Forms che utilizza una libreria .NET Standard non verrà eseguita in un dispositivo Windows 10 Mobile. Le applicazioni Xamarin.Forms continuano a essere eseguite sul desktop di Windows 10, versioni 10.0.16299.0 e successive.

In Xamarin.Forms è supportato l'anteprima per le piattaforme [Mac](~/xamarin-forms/platform/other/mac.md), [WPF](~/xamarin-forms/platform/other/wpf.md), [GTK](~/xamarin-forms/platform/other/gtk.md)e [Tizen](~/xamarin-forms/platform/other/tizen.md) .
