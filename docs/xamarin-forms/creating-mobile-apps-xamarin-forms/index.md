# <a name="creating-mobile-apps-with-xamarinforms-book"></a>Creazione di App per dispositivi mobili con xamarin. Forms libro

[![Scaricare esempio](~/media/shared/download.png) Scaricare l'esempio](https://github.com/xamarin/xamarin-forms-book-samples)

<img src="images/cover-sml.png" title="Libro Creazione di app per dispositivi mobili con Xamarin.Forms" align="left" />Il libro *creazione di app per dispositivi mobili con Novell. Forms* di Charles Petzold è una guida per apprendere come scrivere applicazioni Novell. Forms. L'unico prerequisito è la conoscenza del C# linguaggio di programmazione. Il libro offre un'esplorazione estesa all'interfaccia utente di xamarin. Forms e illustra anche l'animazione, MVVM, trigger, comportamenti, i layout personalizzati, renderer personalizzati e altro ancora.

Il libro è stato pubblicato nella primavera del 2016 e non è stato aggiornato da quel momento. C'è molto del libro che rimane prezioso, ma parte del [materiale è obsoleta](#ways-in-which-the-book-is-outdated)e alcuni argomenti non sono più completamente corretti o completi.

## <a name="ways-in-which-the-book-is-outdated"></a>Modi in cui il libro è obsoleto

Dalla pubblicazione della *creazione di app per dispositivi mobili con Novell. Forms*, sono state aggiunte diverse nuove funzionalità a Novell. Forms. Queste nuove funzionalità sono descritte in singoli articoli nella documentazione di [Novell. Forms](/xamarin/) .

Altre modifiche hanno causato una parte del contenuto del libro obsolete:

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>Le librerie .NET standard 2.0 hanno sostituito le librerie di classi portabile

Un'applicazione xamarin. Forms in genere utilizza una libreria per condividere codice tra piattaforme diverse. Ciò era in origine, una libreria di classi portabile (PCL). Esistono molti riferimenti alle librerie di classi portabili in tutto il libro e i riepiloghi del capitolo.

La libreria di classi portabile è stata sostituita con una libreria .NET Standard 2,0, come descritto nell'articolo [supporto per la .NET Standard 2,0 in Novell. Forms](~/xamarin-forms/internals/net-standard.md). Tutto il [codice di esempio](https://github.com/xamarin/xamarin-forms-book-samples) del libro è stato aggiornato per usare le librerie di .NET standard 2,0.

La maggior parte delle informazioni del libro riguardanti il ruolo della libreria di classi portabile rimane invariato per una libreria .NET Standard 2.0. Una differenza è che solo una libreria di classi Portabile ha un valore numerico "profilo". Inoltre, esistono alcuni vantaggi per le librerie .NET Standard 2.0. Il capitolo 20, [Async e i/o di file](summaries/chapter20.md) , ad esempio, descrive come usare le piattaforme sottostanti per l'esecuzione di operazioni di i/o di file. Ciò non è più necessario. La libreria .NET Standard 2,0 supporta le classi [System.io](xref:System.IO) note per tutte le piattaforme Novell. Forms.

La libreria .NET Standard 2,0 consente inoltre alle applicazioni Novell. Forms di usare [`HttpClient`](xref:System.Net.Http.HttpClient) per accedere ai file su Internet anziché [`WebRequest`](xref:System.Net.WebRequest) o altre classi.

### <a name="the-role-of-xaml-has-been-elevated"></a>È stato portato il ruolo di XAML

La *creazione di app per dispositivi mobili con Novell. Forms* inizia con la descrizione della scrittura di applicazioni C#Novell. Forms tramite. Il Extensible Application Markup Language (XAML) non è stato introdotto fino al [capitolo 7. XAML rispetto al codice](summaries/chapter07.md).

XAML è ora un ruolo di maggior in xamarin. Forms. I modelli di soluzione xamarin. Forms distribuiti con Visual Studio creano i file di paging basato su XAML. Uno sviluppatore che usa xamarin. Forms deve acquisire familiarità con XAML appena possibile. La sezione [Extensible Application Markup Language (XAML)](~/xamarin-forms/xaml/index.yml) della documentazione di Novell. Forms contiene diversi articoli su XAML per iniziare.

### <a name="supported-platforms"></a>Piattaforme supportate

Xamarin. Forms non supporta più Windows 8.1 e Windows Phone 8.1.

Il libro a volte fa riferimento al _Windows Runtime_. Questo è un termine che comprende l'API di Windows usata nelle diverse versioni di Windows e Windows Phone. Le versioni più recenti di xamarin. Forms si limita al supporto della piattaforma Windows universale, ovvero l'API per Windows 10 e Windows 10 Mobile.

Una libreria .NET Standard 2.0 non supporta tutte le versioni di Windows 10 Mobile. Pertanto, un'applicazione xamarin. Forms usando una libreria .NET Standard non verrà eseguito in un dispositivo Windows 10 Mobile. Le applicazioni xamarin. Forms continuano l'esecuzione in versioni 10.0.16299.0 desktop di Windows 10 e versioni successive.

Novell. Forms supporta l'anteprima per le piattaforme [Mac](~/xamarin-forms/platform/other/mac.md), [WPF](~/xamarin-forms/platform/other/wpf.md), [GTK #](~/xamarin-forms/platform/other/gtk.md)e [Tizen](~/xamarin-forms/platform/other/tizen.md) .
